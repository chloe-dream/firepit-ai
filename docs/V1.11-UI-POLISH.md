# V1.11 — Toolbar Icons, Tab/Toolbar Cohesion, Loading Spinner

**Status:** in progress 2026-05-09. Small UX polish cycle on top of V1.1. No functional changes.

## Goal

Three connected polish moves:

1. **Toolbar with icons.** Built-in toolbar buttons (Rekindle, Resume, Explorer, Shell) get a glyph + label, not just text. Quick-links get a small generic link glyph.
2. **Active tab + toolbar = one visual surface.** The active tab's top color matches the toolbar background, so the tab visually flows down into the toolbar. Inactive tabs sit on the darker caption row; the active one is "lifted" onto the toolbar surface.
3. **Initial console-load spinner.** While WebView2 is initializing and the agent is spawning, a small animated spinner sits in the center of the tab content area instead of the static "Igniting {project}…" text.

---

## Scope

**In:**
- `Resources/Icons.xaml` — `Geometry` resources for `IconRekindle`, `IconResume`, `IconExplorer`, `IconShell`, `IconLink`.
- `Views/TabToolbar.xaml` — new button template (icon + label), surface color matches active-tab body.
- `Views/TabToolbar.xaml.cs` — quick-link buttons rendered with a generic link glyph (named-icon registry deferred to V1.12).
- `MainWindow.xaml` — active-tab `Bd` background color matches toolbar; gold accent moves from bottom underline to a top sliver that reads as "the tab pulls up onto the toolbar surface".
- `Views/SessionTab.cs` — replace the initial `_statusText` element with an animated spinner during the period from session start until WebView2 is initialized + child process spawned. `_statusText` stays for fatal-error display.

**Out:**
- Quick-link icon registry by name (`github`, `fishbowl`, …) — V1.12.
- Loading spinner during Rekindle (current `_statusText` is removed before rekindle re-init; rekindle reuses same spinner if cleanly wired).
- Any toolbar-layout shift (still a horizontal strip with quick-link region on the right).

## Design decisions

1. **Cohesion color: continuous body tone.** Active tab + toolbar + terminal area all share `#1A1612`. No "lift" tone — the active tab visually flows straight through the toolbar into the terminal as one contiguous surface. Inactive tabs stay on the darker caption row (`#15110D`).

2. **Gold accent: top sliver on active tab.** Current `BorderThickness="0,0,0,2"` gold underline becomes `BorderThickness="0,2,0,0"`. Reads as the active tab marking its column from the top.

3. **No rounded tab corners.** Square corners (`CornerRadius="0"`) for a more terminal/Edge-modern feel.

3. **Toolbar bottom border stays.** Thin `#332B22` line below the toolbar separates toolbar from terminal viewport. Without it the toolbar floats and the eye loses where input begins.

4. **Spinner geometry: 3/4-circle arc, 1.5px stroke, brand warm-grey, ~24px diameter.** Rotating with a `DoubleAnimation` 0→360° over 900ms, `RepeatBehavior="Forever"`. Stops + removes from visual tree on completion.

5. **Icons live in a `ResourceDictionary`, not inline.** `Geometry` is a freezable type and is safely shareable across `<Path Data="{StaticResource ...}" />` references. Avoids the pitfall called out in `CaptionButtons.xaml` (UIElement Setter.Value is shared per-style and disappears between consumers).

---

## Acceptance

- Toolbar buttons show a glyph next to each label; hover state colors both the glyph stroke and the label foreground.
- Active tab and toolbar share the same background color; eye reads tab + toolbar as one continuous surface; inactive tabs sit on a visibly darker caption row.
- Gold sliver appears at the top of the active tab.
- Opening a project shows a spinning indicator during agent boot; it disappears once the terminal mounts and the agent's first output paints.
- No regressions: tab-close, picker, settings gear, system buttons, quick-link clicks all still work; Rekindle / Resume Last cycle behaves as before.
